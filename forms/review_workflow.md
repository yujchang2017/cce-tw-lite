# 審核員 SOP (v2 — 改編 / 試教 分流版)

> 適用對象: 2-3 位審核員
> 目標: 每件 ≤ 10 分鐘,每週一上午統一處理當週新提交

## 每週流程

### 週一上午 09:00-11:00 (集中審核時段)

1. 打開 Google Sheet (連結見維護者群組)
2. 篩選 `狀態 = pending` 的列
3. 從上往下逐筆審 (按「提交類型」欄分類處理可加速)

### 單筆審核 (≤ 10 分鐘)

#### 第 1 步:看「提交類型」欄

- A (只改編) → 跳到「A 類型 checklist」
- B (只試教) → 跳到「B 類型 checklist」
- C (改編+試教) → 跑全部 checklist

---

#### A 類型 checklist (只改編)

- [ ] **基本完整性** (1 分鐘)
  - 教案編號存在於系統 (查 packages.json)
  - Section 1、Section 2 必填都填了
  - 三個同意書都勾了 (肖像權勾「無相關照片」可接受)
- [ ] **改編內容品質** (5 分鐘)
  - 「改編簡述」有具體描述,不是「優化、改善、調整」這類空話
  - 「改了哪些部分」勾選與「改編簡述」一致
  - 「為什麼這樣改」展現對學生的理解 (不是 ChatGPT 套話)
  - 若有上傳改編教案檔 → 開檔比對是否真有改動
- [ ] **填寫決定** (1 分鐘)
  - 在「狀態」欄選一個值
  - 「審核員」、「審核日期」、「審核備註」填值

#### B 類型 checklist (只試教)

- [ ] **基本完整性** (1 分鐘)
  - 教案編號存在
  - Section 1、Section 3 必填都填了
  - 三個同意書都勾了
- [ ] **照片** (2 分鐘)
  - 開連結看 3-10 張試教照片
  - 確認無學生臉部清晰可辨識,或老師已聲明肖像權
  - 確認與「氣候變遷教育」相關 (排除完全無關的 spam)
- [ ] **省思品質** (3 分鐘)
  - 教學省思 ≥ 500 字
  - 有具體描述「學生在哪裡有感、哪裡卡住、下次會怎麼改」
  - 不是 ChatGPT 通用廢話
- [ ] **填寫決定**

#### C 類型 checklist (改編+試教)

- [ ] **基本完整性** (1 分鐘) — Section 1、2、3 都要填
- [ ] **照片** (2 分鐘) — 同 B 類型
- [ ] **改編內容** (3 分鐘) — 同 A 類型
- [ ] **省思品質** (3 分鐘) — 同 B 類型
- [ ] **C 類型加分項**: 改編內容與試教省思有對應關係 (例如:省思提到「我把第三段改成在地案例,學生反應很好」)
- [ ] **填寫決定** — C 類型優先 approve (最高品質)

---

## 退回 / 拒絕的常見原因

| 類型 | 常見退回原因 |
|---|---|
| A | 改編簡述太空泛、沒附改編檔且文字看不出實質改動 |
| B | 試教照片不足、省思 < 500 字、Section 3 漏填某題 |
| C | 改編與試教明顯不一致 (改編寫一套、試教寫另一套) |

## 標準回覆模板

### needs-revision

```
{老師姓名} 您好,

您提交的《{教案編號}》(類型: {A/B/C}) 已收到,但需要補充以下資訊:

- [ ] (依實際勾選)
- [ ] 試教照片不足 3 張
- [ ] 教學省思未達 500 字

```

## 週一下午 / 週二: 維護者批次合併

由系統維護者 (1 人) 執行,把 approved 的提交合到 cce-teaching-packages repo。

### 不同類型的合併方式

| 類型 | 合併位置 | 檔名範例 |
|---|---|---|
| A (只改編) | `packages/level-{lvl}/{keyId}-remixes/teacher-{name}-{ts}/` | `adaptation.md`, `worksheet.pdf` |
| B (只試教) | `packages/level-{lvl}/{keyId}-trials/teacher-{name}-{ts}/` | `reflection.md`, `photos/` |
| C (改編+試教) | `packages/level-{lvl}/{keyId}-remixes/teacher-{name}-{ts}/` (含試教資料) | `adaptation.md`, `reflection.md`, `photos/` |

### 步驟

1. 在本機 cd 到 `community_cce_tw_lite/`
2. 確保有最新 fine-grained PAT (僅給 cce-teaching-packages repo,Contents+PR write,90 天到期)
3. 執行 `node scripts/weekly-merge.mjs`
   - script 讀 Sheet 中 `approved` 但 `PR 連結` 為空的列
   - 依「提交類型」決定資料夾結構
   - download 老師上傳的檔案
   - 開 branch + commit + open PR
   - 在 Sheet 寫回 PR 連結
4. 維護者人工 review PR 後 merge
5. GitHub Actions 自動 redeploy → 5 分鐘後上線

> ℹ️ **`weekly-merge.mjs` 尚未實作** (Week 3 任務)。Phase 0 第一個月可手動處理。

## 升級徽章 (月底批次)

每月最後一個工作日:

1. 開 Sheet,計算每位老師的 approved 件數,**依類型分別計**
2. 對照徽章規則 (見 [PLAN.md](../PLAN.md) §徽章規則)
3. 寫信通知升級者
4. 在年會公開頒發實體徽章

### 徽章規則 (v2)

| 徽章 | 規則 |
|---|---|
| 🌱 新芽 | 提交 ≥ 1 件 (任何類型) |
| 🌳 青樹 | 改編 ≥ 1 + 試教 ≥ 1 (可分屬不同教案) |
| 🏔️ 氣候教育師 | 6 大主題各有 ≥ 1 件「改編+試教 (C 類)」或 (改編+試教各 1 件) |

> 💡 C 類型因含金量最高,計分加倍 — 1 件 C 類可同時抵 1 件 A + 1 件 B。
